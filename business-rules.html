Business Rules & Thresholds

How the Take 5 Close-Out App handles submissions, variances, and alerts

These are the current business rules built into the app. All values can be adjusted — let Elliott know if any thresholds or behaviors need to change.

Variance & Flagging

Variance Threshold $15
Any close-out where the difference between POS total and (batches minus tips) exceeds $15 is automatically flagged. The manager must select a reason before submitting.

Question: What should happen when a variance is flagged? Options: immediate notification to office, hold submission for approval, or just log the reason and move on?

Variance Reason Options 6 categories
When flagged, manager selects from: Tips not backed out, Fleet card mismatch, Split batch, Manual entry error, Refund after close, or Other.
These categories can be added to or modified.

Deposit Tracking

Overdue Deposit Alert 48 hours
If a nightly close-out has cash to deposit and no deposit slip has been submitted within 48 hours, the store appears in the overdue deposit alert email.
Deposit Alert Email 2:00 PM ET daily
Automated email listing all stores with overdue deposits. Sent to office/admin recipients.

Submission Rules

One Per Store Per Day 1 nightly + 1 deposit
Each store can submit one nightly close-out and one deposit slip per date. Duplicates are blocked with a clear message directing the user to resubmit through History.
Resubmit / Corrections Soft delete
When a manager resubmits for a date, the old submission is marked as "superseded" (not deleted). Both versions are kept for audit purposes with timestamps and who made the change.
Future Dates Blocked
Cannot submit a close-out for a future date.
Old Date Warning 30 days
Submitting for a date more than 30 days ago triggers a confirmation dialog. The manager can still proceed.
Submission Queue Last 7 days
The manager's home screen shows the last 7 days with green (submitted) or orange (missing) indicators for both nightly and deposit. Tap a missing one to start submitting.

Automated Emails

Nightly Compliance Summary 10:00 PM ET daily
Lists which stores submitted their nightly close-out and which didn't. Includes flagged variances.
Overdue Deposit Alert 2:00 PM ET daily
Lists stores where cash hasn't been deposited within 48 hours of the close-out.

Review Workflow

Submission Statuses 5 states
Pending Review → default when submitted
Reviewed → office staff has checked it
Flagged → variance or issue needs attention
Resolved → flagged issue has been addressed
Superseded → replaced by a resubmission

Technical Defaults

Default Terminals Per Store 4
The app defaults to 4 batch terminal inputs. Each store's actual terminal count is set in the database (e.g., Asheville 1 has 3).
Photo Compression 70% JPEG
Receipt photos are compressed to 70% quality before upload to reduce storage and speed up OCR processing. This maintains readability while keeping file sizes small.
Database Backups Daily at midnight ET
Automated nightly database backup via GitHub Actions. 30-day retention.

Open Questions

Variance actions: What should happen when a variance exceeds $15? Should the office be notified immediately? Should the submission require approval? Or is selecting a reason sufficient?

Any thresholds to adjust? The $15 variance flag, 48-hour deposit window, 7-day queue display, and 30-day old-date warning can all be changed. Let me know what works for your operations.
